Class Mcontroller

Description
Description | Descendents | Vars (details) | Methods (details)

Mcontroller has several control functions:

  1. Allow control to flow only if appropriate permission conditions are met.
  2. Branch URLs of the form className=...&action=... to the corresponding class method.
  3. Serve as a superclass to be extended with seamless inheritance from Mmoel and Mview.

		control (line 117)
	
 
	
	

control() decides which method of which class is executed.


It is typically called from index.php  after a control class is extended from Mcontroller:
  $mcc = new MyControlClass;
  $mcc->control();
className denotes the class name and can be presented: 
  1. in the URL as className=...
  2. as the first part of path info
  3. as a hidden field in a form.
 the method is denoted by the word 'action' and can be presented:
 
  1. in the URL as action=...
  2. as the second part of path info
  3. as a hidden field in a form.
  The class Abc will be automatically loaded from Abc.class.php  if it is not already loaded and there is such a file.
if called from within a controller, dispatiching is redirected and args are (re)placed in $_REQUEST
  e.g.
    $this->control("Authors", "listBooks", array('authorId' => 2,))
  will execute as if from a url ?className=Authors&action=listBooks&authorId=2
  this re-dispatches the action directly (without a web redirect)

  so
    $this->control("Authors", "listBooks", array('authorId' => 2,))
    $this->control("Authors", "listBooks", array('authorId' => 3,))
  might have the effect of rendering the results of both actions consecutively on the same page
use redirect($className = null, $action = null, $args = null)
  to replace the current action with a new one like traditional frameworks
control calls permit() before it allows execution to take place. see below.
if the extended object has a before() method it is called first...likewise after().
  onceBefore() and onceAfter() are called only once per run.
